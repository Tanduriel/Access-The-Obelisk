## Notes

- Across the Obelisk does not appear to be open source, so this project will use a separate mod.
- The game already has BepInEx-based community mods, which is a good sign for feasibility.
- Because the user is not familiar with the game, tutorial and mechanics text extraction should be prioritized during analysis.
- `InputController` already routes arrow/WASD navigation to active game screens, so the first accessibility features should announce existing focus/state instead of replacing controls.

## Architecture Decisions

- Use a central accessibility state manager. Reason: many screens share the same navigation keys and `InputController.DoMovementVector()` already branches across many active managers.
- First feature should be low-risk: startup announcement plus tutorial popup announcement.
- Main menu focus announcements should come next, using `MainMenuManager` and visible `MenuButton`/`BotonGeneric` text.
